# บทที่ 3 — มุมมองระดับสูงของฟังก์ชันและการเชื่อมต่อคอมพิวเตอร์

## 3.2 ฟังก์ชันของคอมพิวเตอร์ (Computer Function)

ฟังก์ชันพื้นฐานของคอมพิวเตอร์คือการ **ดำเนินการโปรแกรม** ซึ่งประกอบด้วยชุดคำสั่ง (Instructions) ที่เก็บอยู่ใน Memory

---

### 3.2.1 การดึงและดำเนินการคำสั่ง (Instruction Fetch and Execute)

วงจรคำสั่ง (Instruction Cycle) ประกอบด้วยขั้นตอนหลัก ดังนี้

#### ขั้นตอนในวงจรคำสั่ง

| ตัวย่อ | ชื่อขั้นตอน | คำอธิบาย |
|---|---|---|
| **iac** | Instruction Address Calculation | คำนวณที่อยู่ของคำสั่งถัดไป |
| **if** | Instruction Fetch | ดึงคำสั่งจาก Memory เข้าสู่ Processor |
| **iod** | Instruction Operation Decoding | วิเคราะห์คำสั่งเพื่อระบุประเภทของการดำเนินการ |
| **oac** | Operand Address Calculation | คำนวณที่อยู่ของ Operand (หากจำเป็น) |
| **of** | Operand Fetch | ดึง Operand จาก Memory หรือ I/O |
| **do** | Data Operation | ดำเนินการตามคำสั่ง |
| **os** | Operand Store | เขียนผลลัพธ์กลับไปยัง Memory หรือ I/O |

> **ตัวอย่าง:** คำสั่ง `ADD A,B` บน PDP-11 จะมีลำดับสถานะดังนี้:  
> `iac → if → iod → oac → of → do → oac → os`

---

### 3.2.2 การขัดจังหวะ (Interrupts)

**ปัญหาหากไม่มี Interrupt:** โปรเซสเซอร์ต้องรอให้การดำเนินการ I/O เสร็จสมบูรณ์ก่อน ซึ่งทำให้เสียเวลาเปล่า เพราะอุปกรณ์ I/O ทำงานช้ากว่า CPU มาก

**แนวคิดของ Interrupt:**
- ขณะที่การดำเนินการ I/O กำลังดำเนินอยู่ CPU สามารถประมวลผลคำสั่งอื่นในโปรแกรมของผู้ใช้ได้ต่อเนื่อง
- เมื่ออุปกรณ์ I/O พร้อมรับข้อมูลชุดถัดไป โมดูล I/O จะส่งสัญญาณ **Interrupt Request** ไปยัง Processor
- Processor จะหยุดโปรแกรมปัจจุบันชั่วคราว แล้วโอนการควบคุมให้ **Interrupt Handler**
- เมื่อ Interrupt Handler ทำงานเสร็จ Processor จะกลับมาทำงานในโปรแกรมเดิมต่อ

#### ลำดับขั้นตอนเมื่อเกิด Interrupt

1. โปรแกรมผู้ใช้เรียก `WRITE` → โอนไปยัง I/O Program
2. I/O Program ส่งคำสั่ง I/O ไปยังอุปกรณ์
3. ควบคุมกลับมายังโปรแกรมผู้ใช้ — CPU ทำงานต่อไปได้ทันที
4. อุปกรณ์พร้อม → ส่ง Interrupt Request
5. CPU บันทึกสถานะปัจจุบัน → ทำงาน Interrupt Handler
6. Interrupt Handler สิ้นสุด → กลับสู่โปรแกรมผู้ใช้

---

### 3.2.3 ฟังก์ชัน I/O (I/O Function)

โมดูล I/O ทำหน้าที่เป็นตัวกลางระหว่าง CPU/Memory กับอุปกรณ์ภายนอก โดยมีบทบาทหลัก ได้แก่

- **ควบคุมและกำหนดจังหวะ (Control & Timing):** ประสานการถ่ายโอนข้อมูลระหว่างอุปกรณ์ที่ทำงานต่างความเร็วกัน
- **สื่อสารกับ CPU:** รับคำสั่งและรายงานสถานะ
- **สื่อสารกับอุปกรณ์:** ส่งคำสั่งและรับข้อมูล
- **บัฟเฟอร์ข้อมูล (Data Buffering):** ชดเชยความต่างของอัตราการถ่ายโอนข้อมูล
- **ตรวจสอบข้อผิดพลาด (Error Detection)**

# บทที่ 3 — มุมมองระดับสูงของฟังก์ชันและการเชื่อมต่อคอมพิวเตอร์

## 3.3 โครงสร้างการเชื่อมต่อ (Interconnection Structures)

ส่วนประกอบหลักของคอมพิวเตอร์ (CPU, Memory, I/O) ต้องสื่อสารกันผ่านเส้นทางการเชื่อมต่อ โดยประเภทและปริมาณข้อมูลที่แลกเปลี่ยนขึ้นอยู่กับลักษณะของแต่ละองค์ประกอบ

---

### ประเภทของการส่งสัญญาณ

| จาก → ไปยัง | ประเภทข้อมูลที่ส่ง |
|---|---|
| **Memory → CPU** | คำสั่ง (Instruction) และข้อมูล (Data) |
| **CPU → Memory** | ข้อมูลสำหรับเขียน (Write Data) |
| **I/O → CPU** | ข้อมูลจากอุปกรณ์ภายนอก และสัญญาณ Interrupt |
| **CPU → I/O** | คำสั่งควบคุม และข้อมูลที่ต้องการส่งออก |
| **I/O ↔ Memory** | การถ่ายโอนข้อมูลโดยตรง (DMA — Direct Memory Access) |

---

### รูปแบบโครงสร้างการเชื่อมต่อ

#### 1. Bus Structure (โครงสร้างแบบบัส)
- ส่วนประกอบทุกชิ้นเชื่อมต่อกันผ่านสายสัญญาณร่วม (Shared Bus)
- ข้อดี: ต้นทุนต่ำ ง่ายต่อการขยาย
- ข้อเสีย: คอขวด เพราะครั้งละอุปกรณ์เดียวเท่านั้นที่ใช้บัสได้

#### 2. Point-to-Point Interconnect (การเชื่อมต่อแบบจุดต่อจุด)
- อุปกรณ์แต่ละคู่มีเส้นทางสื่อสารเป็นของตัวเอง
- ข้อดี: แบนด์วิดท์สูง ลดการชนกันของข้อมูล
- ตัวอย่าง: Intel QPI, PCIe

# บทที่ 3 — มุมมองระดับสูงของฟังก์ชันและการเชื่อมต่อคอมพิวเตอร์

## 3.4 การเชื่อมต่อแบบบัส (Bus Interconnection)

**บัส (Bus)** คือเส้นทางการสื่อสารร่วมที่เชื่อมต่อองค์ประกอบสองชิ้นขึ้นไป โดยสัญญาณที่ส่งผ่านบัสจะถูกรับโดยทุกอุปกรณ์ที่เชื่อมต่ออยู่

---

### ประเภทของสายบัส

| ประเภท | หน้าที่ |
|---|---|
| **Data Bus** | ส่งข้อมูลระหว่างโมดูล ความกว้าง (Bus Width) กำหนดจำนวน bit ที่โอนได้ในครั้งเดียว เช่น 32-bit หรือ 64-bit |
| **Address Bus** | ระบุตำแหน่ง (Address) ของ Memory หรืออุปกรณ์ I/O ที่ต้องการสื่อสาร |
| **Control Bus** | ส่งสัญญาณควบคุม เช่น Read/Write, Clock, Interrupt Request, Bus Request |

---

### ปัญหาของบัสร่วม (Shared Bus)

- ยิ่งอุปกรณ์เชื่อมต่อมาก ความล่าช้าในการแพร่กระจายสัญญาณ (Propagation Delay) ยิ่งสูง
- เกิดคอขวด (Bottleneck) เมื่ออุปกรณ์หลายชิ้นต้องการใช้บัสพร้อมกัน
- แบนด์วิดท์รวมของระบบถูกจำกัดโดยบัสเส้นเดียว

---

### การใช้หลายบัส (Multiple Bus Hierarchy)

เพื่อแก้ปัญหาคอขวด ระบบสมัยใหม่แบ่งบัสออกเป็นลำดับชั้น เช่น

```
CPU ─── Local Bus ─── Cache/Bridge
                          │
                     System Bus (Memory Bus)
                          │
                    Expansion Bus ─── I/O Devices
```

- **Local Bus:** ความเร็วสูง เชื่อมระหว่าง CPU กับ Cache และ Local Memory
- **System Bus:** เชื่อมต่อ CPU Bridge กับ Main Memory
- **Expansion Bus:** ความเร็วต่ำกว่า ใช้สำหรับอุปกรณ์ I/O ทั่วไป

---

### การควบคุมบัส (Bus Arbitration)

เมื่ออุปกรณ์หลายชิ้นต้องการใช้บัสพร้อมกัน ต้องมี **Bus Arbiter** ตัดสินว่าใครได้ใช้งาน โดยมีสองแบบ:

| แบบ | คำอธิบาย |
|---|---|
| **Centralized Arbitration** | มี Arbiter ศูนย์กลางตัวเดียวตัดสินใจ |
| **Distributed Arbitration** | อุปกรณ์แต่ละชิ้นร่วมกันตัดสินใจผ่านโปรโตคอล |

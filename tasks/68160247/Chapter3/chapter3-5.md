# บทที่ 3 — มุมมองระดับสูงของฟังก์ชันและการเชื่อมต่อคอมพิวเตอร์

## 3.5 การเชื่อมต่อแบบจุดต่อจุด (Point-to-Point Interconnect)

เนื่องจากข้อจำกัดของบัสร่วม (Shared Bus) สถาปัตยกรรมสมัยใหม่จึงหันมาใช้การเชื่อมต่อแบบ **จุดต่อจุด (Point-to-Point)** ซึ่งแต่ละคู่ของอุปกรณ์มีเส้นทางสื่อสารเป็นของตัวเอง

ตัวอย่างที่สำคัญได้แก่ **Intel QuickPath Interconnect (QPI)**

---

### สถาปัตยกรรมแบบ QPI

QPI แบ่งโครงสร้างออกเป็น **4 ชั้น (Layers)**:

---

### 3.5.1 ชั้นกายภาพ (QPI Physical Layer)

- เป็นชั้นล่างสุด ทำหน้าที่ส่งบิตข้อมูลจริงๆ ผ่านสายสัญญาณ
- ใช้ **Lane** แต่ละเส้นส่งข้อมูลแบบ Differential Signaling (คู่สาย + และ −)
- QPI มี **20 Lane** ต่อทิศทาง รวมกับสัญญาณนาฬิกา (Clock) 1 Lane = 84 สาย
- รองรับการส่งข้อมูล 2 บิตต่อ Clock Cycle (Double Data Rate)

---

### 3.5.2 ชั้นลิงก์ (QPI Link Layer)

- จัดการการรับส่งข้อมูลเป็น **Flit (Flow Control Unit)** ขนาด 80 บิต
- ทำหน้าที่ควบคุมการไหลของข้อมูล (Flow Control) และตรวจสอบความผิดพลาด (Error Control)
- ใช้ CRC (Cyclic Redundancy Check) เพื่อตรวจสอบความถูกต้องของข้อมูล

---

### 3.5.3 ชั้นเส้นทาง (QPI Routing Layer)

- กำหนดเส้นทางที่ข้อมูลแต่ละ Packet จะเดินทางผ่านระบบ
- ในระบบ Multi-Processor ชั้นนี้เลือกเส้นทางที่เหมาะสมเพื่อส่งข้อมูลจาก Node หนึ่งไปยังอีก Node หนึ่ง

---

### 3.5.4 ชั้นโปรโตคอล (QPI Protocol Layer)

- เป็นชั้นบนสุด กำหนดกฎในการแลกเปลี่ยนข้อมูลระหว่างอุปกรณ์
- รองรับโปรโตคอลสำหรับ Cache Coherency (ความสอดคล้องของแคช) และ Memory Access
- ข้อมูลถูกจัดรูปแบบเป็น **Packet** ก่อนส่ง ซึ่งประกอบด้วย Header และ Data

---

### ข้อดีของ Point-to-Point เทียบกับ Shared Bus

| คุณสมบัติ | Shared Bus | Point-to-Point (QPI) |
|---|---|---|
| แบนด์วิดท์ | จำกัด | สูงมาก (ไม่แชร์กัน) |
| ความหน่วง | สูง (ต้องรอ Arbitration) | ต่ำ |
| การขยายระบบ | ยาก | ง่าย |
| ความซับซ้อน | ต่ำ | สูงกว่า |
